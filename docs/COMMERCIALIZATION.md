# 商业化说明

## 当前选择的商业化形态

本仓库当前采用的商业化形态是：**GPL v3 自托管单租户部署 + 有偿支持 / 实施服务**。

这意味着：

- 仓库继续以 GPL v3 发布
- 支持向客户提供部署、集成、培训、迁移、调优、运维支持等有偿服务
- 支持客户在其自有环境中以单租户方式运行本产品

## 当前不支持的商业化形态

以下形态**不属于当前主线产品承诺**：

- 公网多租户托管 SaaS
- 以闭源专有许可证重新分发当前仓库
- 未经额外法律与产品设计收口的账号 / 计费 / 组织空间模式

## 交付边界

当前可支持的交付边界为：

1. 作者或团队在本地 / 受信任网络 / 自有服务器中部署
2. Dashboard 作为只读观察面使用
3. CLI / Codex 作为实际写作主链路使用
4. 通过 Basic Auth 对非本地访问做最小访问保护

## 质量验收边界

仓库内 CI 的 `app-release-gate.yml` 只负责验证：

- `rag verify` 命令链路可执行
- benchmark 产物契约可被正确读取
- 缺失 benchmark 时会 fail closed

这**不是客户级质量承诺本身**。

对于实际商业交付，验收应增加一条客户环境准入规则：

1. 由客户或交付方在目标项目中准备 `.webnovel/codex/rag-benchmark.json`
2. 在目标部署环境执行：

```bash
python -X utf8 webnovel-writer/scripts/webnovel.py \
  --project-root "<PROJECT_ROOT>" \
  codex rag verify \
  --report json
```

3. 只有当该项目自己的 benchmark 报告通过时，才将其视为该客户环境中的交付验收通过

## 销售与承诺口径

当前可以销售的是：

- 自托管部署服务
- 导入 / 迁移服务
- 写作工作流集成服务
- 运维与支持服务

当前不应销售的是：

- “现成可公网托管的 SaaS 平台”
- “已经对任意小说项目证明了统一准确率 / 性能 SLA 的平台能力”

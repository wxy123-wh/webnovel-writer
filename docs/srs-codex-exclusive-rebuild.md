## 2. 项目背景与目标

### 2.1 背景

当前项目由 Claude 插件体系迁移而来，代码、配置、目录与文档中仍残留大量 Claude 兼容逻辑，导致 Codex 体验不一致、维护成本高、功能边界不清晰。

### 2.2 总体目标

1. 将系统彻底改造成 **Codex 专属**，不再兼容 Claude Code。
2. 新增 **工作区专属 Skill 库**，支持自由新增、启用、禁用，仅在小说工作区加载。
3. 新增 `设定集` 与 `双纲同屏`（总纲+细纲）核心界面。
4. 支持 Codex 从设定集中自动抽离并生成 **设定词典**。
5. 支持在总纲中选中文字后右键拆分，执行无修饰扩写并分段写入细纲。
6. 细纲由全部场景化拆分片段构成，系统需记录拆分位置并保证顺序稳定。
7. 支持对已拆分区域重新拆分，按“选区更小/更大”规则自动回退后再拆分。
8. 全部文字区域支持选中文字右键唤起 Codex 协助修改。
9. 同步更新全部相关文档，形成统一 Codex 叙事。

## 3. 范围定义

### 3.1 范围内（In Scope）

1. Codex 专属运行时、目录、入口、配置改造。
2. Claude 插件相关代码路径与文档路径下线或迁移。
3. 工作区 Skill 库管理能力（新增、启用、禁用、按工作区加载）。
4. 设定集页签与设定词典自动抽离能力。
5. 总纲/细纲双栏同屏、右键拆分、顺序化落盘、重拆回退规则。
6. 全局文本右键唤起 Codex 协助修改。
7. README 与 docs 全量同步改写。

### 3.2 范围外（Out of Scope）

1. 对 Claude Code 的兼容层与双端策略保留。
2. Marketplace 的 Claude 插件发版链路延续。
3. 纲要历史版本快照与多版本回溯能力。

## 4. 角色与使用场景

| 角色 | 职责 |
|---|---|
| 作者（主用户） | 维护设定、总纲与细纲，触发拆分/重拆，确认修改结果 |
| Codex 助手 | 执行设定抽离、总纲拆分、文本协助修改，输出可追溯结果 |
| 维护者（开发） | 负责功能实现、配置迁移、文档更新、质量保障 |

## 5. 核心术语

| 术语 | 定义 |
|---|---|
| 工作区（Workspace） | 当前 Codex 会话对应目录，作为技能与项目绑定边界 |
| 小说项目根（Project Root） | 单本小说目录，包含正文、设定集、大纲等内容 |
| 工作区 Skill 库 | 仅在当前小说工作区生效的技能集合 |
| 设定集 | 设定原文集合（世界观、角色、势力、规则等） |
| 设定词典 | 从设定集抽离出的结构化知识条目库 |
| 总纲 | 作者维护的原始纲要文本，作为拆分源 |
| 细纲 | 由总纲拆分后得到的场景化分段纲要，按顺序存储 |
| 拆分锚点 | 总纲中被拆分文本的位置记录（起止偏移/段落索引） |
| 场景片段 | 细纲中的最小段落单元，仅描述一个剧情或画面 |

## 6. 总体约束与原则

1. 平台原则：仅支持 Codex，不保留 Claude 兼容分支。
2. 加载原则：Skill 默认按工作区加载，不污染全局。
3. 双纲原则：系统仅维护两个纲要，`总纲` 与 `细纲`。
4. 拆分原则：拆分结果必须是无修饰扩写，分段写入细纲。
5. 顺序原则：细纲片段按拆分顺序存储，严禁错序。
6. 可追溯原则：设定词典与细纲片段均需可追溯到来源文本。
7. 人机协作原则：作者拥有最终确认权。

## 8. 关键业务规则

| 编号 | 规则 |
|---|---|

| BR-OUT-002 | 总纲是拆分源，细纲是拆分结果，不允许反向覆盖。 |
| BR-SCN-001 | 细纲片段必须无修辞，仅包含剧情动作或画面信息。 |

## 11. 交互与流程需求

### 11.1 设定抽离流程

1. 作者在 `设定集` 页签维护原始设定文档。
2. 作者触发“抽离设定词典”。
3. Codex 输出结构化词典条目并标记冲突。
4. 作者确认冲突后完成词典更新。

### 11.2 总纲拆分到细纲流程

1. 作者进入总纲/细纲同屏界面。
2. 在总纲中选中文字，右键点击“拆分到细纲”。
3. Codex 对选中文本执行无修饰扩写，按段落生成场景化片段。
4. 系统记录拆分锚点并按顺序写入细纲。
5. 写入后在同屏界面显示来源映射，便于对比核验。

### 11.3 重拆回退流程

1. 作者在总纲中重新选中已拆分相关文本。
2. 若新选区小于已拆分范围，系统默认回退整段后重拆。
3. 若新选区大于已拆分范围，系统默认回退所有覆盖片段后重拆。
4. 系统按锚点重新排序并落盘细纲。

### 11.4 全局文本协助修改流程

1. 作者在任意文本区选中文字。
2. 右键点击“唤起 Codex 协助修改”。
3. Codex 输出修改建议与预览。
4. 作者确认后应用修改并记录日志。

## 14. 风险与应对

| 风险 | 影响 | 应对 |
|---|---|---|
| 历史兼容逻辑拆除引发回归 | 核心命令不可用 | 分阶段下线 + 回归测试清单 |
| 拆分锚点记录异常 | 细纲顺序错乱 | 增加落盘前顺序校验与冲突阻断 |
| 无修饰扩写约束不稳定 | 输出偏离预期 | 增加规则校验器与失败重试模板 |
| 全局右键入口一致性不足 | 功能体验不统一 | 统一编辑组件协议与集成测试 |
| 文档改造滞后 | 用户误用旧命令 | 与功能开发并行更新，发布前统一审校 |

## 15. 假设与待确认事项

1. 不提供纲要历史版本功能（按最新规则覆盖式维护）。
2. 仅维护两个纲要文件：`总纲.md` 与 `细纲.md`。
3. 细纲即场景化拆分结果的有序集合，不单独维护“历史场景版本”。
4. 默认继续使用现有目录命名（`设定集/`、`大纲/`、`正文/`）。
5. 本 SRS 为需求基线，详细技术设计在 HLD/LLD 阶段补充。

# 接口说明

> 基线：按仓库代码现状整理（扫描日期 2026-03-27）。

## 全局约定

- CLI 统一入口：`python -X utf8 webnovel-writer/scripts/webnovel.py <tool> ...`
- Dashboard API 默认本地地址：`http://127.0.0.1:8765`
- 写接口与部分读接口使用 `workspace_id + project_root` 做工作区隔离
- 统一错误结构：`{ error_code, message, details, request_id }`
- 当前未启用 token/session 鉴权，建议只在受信任内网使用

## 1. CLI 接口

### 1.1 核心入口（webnovel.py）

| 命令 | 用途 |
|---|---|
| `where` | 输出当前解析到的 `project_root` |
| `preflight` | 运行环境与目录预检 |
| `use` | 绑定工作区指针到目标项目 |
| `dashboard` | 启动 Dashboard 服务 |
| `init` | 初始化小说项目目录 |
| `extract-context` | 提取章节上下文 |
| `migrate codex` | 执行历史路径迁移（支持 `--dry-run`） |
| `status/index/state/rag/context/entity/style/workflow/update-state/backup/archive/consistency-check` | 转发到对应数据模块 |

示例：

```bash
python -X utf8 webnovel-writer/scripts/webnovel.py init ./webnovel-project "你的书名" "题材"
python -X utf8 webnovel-writer/scripts/webnovel.py use "<PROJECT_ROOT>" --workspace-root "<WORKSPACE_ROOT>"
python -X utf8 webnovel-writer/scripts/webnovel.py --project-root "<PROJECT_ROOT>" preflight --format json
python -X utf8 webnovel-writer/scripts/webnovel.py --project-root "<PROJECT_ROOT>" dashboard --port 8765 --no-browser
```

## 2. HTTP API 接口

### 2.1 系统与只读聚合（`dashboard/app.py`）

- 系统：
  - `GET /health`
  - `GET /api/project/root`
  - `GET /api/project/info`
- 实体与关系：
  - `GET /api/entities`
  - `GET /api/entities/{entity_id}`
  - `GET /api/relationships`
  - `GET /api/relationship-events`
  - `GET /api/aliases`
- 章节与审查：
  - `GET /api/chapters`
  - `GET /api/scenes`
  - `GET /api/reading-power`
  - `GET /api/review-metrics`
  - `GET /api/state-changes`
  - `GET /api/checklist-scores`
- 分析与图谱：
  - `GET /api/dashboard/overview`
  - `GET /api/graph`
  - `GET /api/tool-stats`
  - `GET /api/rag-queries`
- 债务与纠错：
  - `GET /api/overrides`
  - `GET /api/debts`
  - `GET /api/debt-events`
  - `GET /api/invalid-facts`
- 文件与事件：
  - `GET /api/files/tree`
  - `GET /api/files/read`
  - `GET /api/events`（SSE）

### 2.2 Runtime（`/api/runtime`）

- `GET /api/runtime/profile`
- `POST /api/runtime/migrate`

用途：返回运行时画像、检查迁移预览、执行迁移并输出报告路径。

### 2.3 Skills（`/api/skills`）

- `GET /api/skills`
- `POST /api/skills`
- `PATCH /api/skills/{skill_id}`
- `POST /api/skills/{skill_id}/enable`
- `POST /api/skills/{skill_id}/disable`
- `DELETE /api/skills/{skill_id}`
- `GET /api/skills/audit`

用途：工作区技能增删改查、启停、审计检索。

### 2.4 Settings（`/api/settings/files` + `/api/settings/dictionary`）

- 文件树与文件读写：
  - `GET /api/settings/files/tree`
  - `GET /api/settings/files/read`
  - `POST /api/settings/files/write`
- 词典抽离与冲突处理：
  - `POST /api/settings/dictionary/extract`
  - `GET /api/settings/dictionary`
  - `GET /api/settings/dictionary/conflicts`
  - `POST /api/settings/dictionary/conflicts/{id}/resolve`

### 2.5 Outlines（`/api/outlines`）

- `GET /api/outlines`
- `POST /api/outlines/split/preview`
- `POST /api/outlines/split/apply`
- `GET /api/outlines/splits`
- `POST /api/outlines/resplit/preview`
- `POST /api/outlines/resplit/apply`
- `POST /api/outlines/order/validate`

用途：双纲读取、拆分预览与落盘、重拆回退、顺序校验。

### 2.6 Edit Assist（`/api/edit-assist`）

- `POST /api/edit-assist/preview`
- `POST /api/edit-assist/apply`
- `GET /api/edit-assist/logs`

用途：选区改写预览、确认应用、日志追溯与版本冲突保护。

### 2.7 Codex Bridge（`/api/codex`）

- `POST /api/codex/split-dialog/open`
- `POST /api/codex/file-edit/open`

用途：在 Dashboard 侧拉起本机 Codex CLI 会话，生成并传递 prompt 文件。

## 4. 常见错误码分层

| 状态码 | 常见场景 |
|---|---|
| `400` | 参数非法、选区非法、时间范围非法 |
| `403` | `workspace_id` 与 `project_root` 不匹配、路径越界 |
| `404` | 项目根不存在、目标资源不存在 |
| `409` | 并发写冲突、版本冲突、顺序冲突 |
| `500` | 文件写入/迁移/服务内部失败 |
| `501` | 运行时能力或外部编辑能力不可用 |

# 接口说明

> 基线：按当前主线代码整理。目标是描述**现行支持的接口面**，而不是历史能力全集。

## 全局约定

- CLI 统一入口：`python -X utf8 webnovel-writer/scripts/webnovel.py <tool> ...`
- Dashboard 默认本地地址：`http://127.0.0.1:8765`
- Dashboard 当前主线以**只读接口**为主，写作动作通过 CLI / Codex 直接编辑文件完成
- 统一错误结构：`{ error_code, message, details, request_id }`
- 可选内置 Basic Auth：`--basic-auth user:password` 或 `WEBNOVEL_DASHBOARD_BASIC_AUTH`

## 1. CLI 接口

### 1.1 统一入口（`webnovel.py`）

| 命令 | 用途 |
|---|---|
| `where` | 输出当前解析到的 `project_root` |
| `preflight` | 运行环境与目录预检 |
| `use` | 绑定工作区指针到目标项目 |
| `dashboard` | 启动 Dashboard 服务 |
| `init` | 初始化小说项目目录 |
| `extract-context` | 提取章节上下文 |
| `migrate codex` | 执行历史路径迁移（支持 `--dry-run`） |
| `codex session start/stop` | 启动 / 停止 Codex 写作会话 |
| `codex index status` | 查询索引状态 |
| `codex rag verify` | 查询 RAG 当前状态 |
| `status/index/state/rag/context/entity/style/workflow/update-state/backup/archive/consistency-check` | 转发到对应数据模块 |

示例：

```bash
python -X utf8 webnovel-writer/scripts/webnovel.py init ./webnovel-project "你的书名" "题材"
python -X utf8 webnovel-writer/scripts/webnovel.py use "<PROJECT_ROOT>" --workspace-root "<WORKSPACE_ROOT>"
python -X utf8 webnovel-writer/scripts/webnovel.py --project-root "<PROJECT_ROOT>" dashboard --port 8765 --no-browser
python -X utf8 webnovel-writer/scripts/webnovel.py --project-root "<PROJECT_ROOT>" codex rag verify --report json
```

## 2. HTTP API 接口

### 2.1 系统与只读聚合（`dashboard/app.py`）

- 系统：
  - `GET /health`
  - `GET /api/project/root`
  - `GET /api/project/info`
- 实体与关系：
  - `GET /api/entities`
  - `GET /api/entities/{entity_id}`
  - `GET /api/relationships`
  - `GET /api/relationship-events`
  - `GET /api/aliases`
- 章节与审查：
  - `GET /api/chapters`
  - `GET /api/scenes`
  - `GET /api/reading-power`
  - `GET /api/review-metrics`
  - `GET /api/state-changes`
  - `GET /api/checklist-scores`
- 分析与图谱：
  - `GET /api/dashboard/overview`
  - `GET /api/graph`
  - `GET /api/tool-stats`
  - `GET /api/rag-queries`
- 债务与纠错：
  - `GET /api/overrides`
  - `GET /api/debts`
  - `GET /api/debt-events`
  - `GET /api/invalid-facts`
- 文件与事件：
  - `GET /api/files/tree`
  - `GET /api/files/read`
  - `GET /api/events`（SSE）

### 2.2 Runtime（`/api/runtime`）

- `GET /api/runtime/profile`

用途：返回运行时画像（只读）。迁移写操作已迁移到 CLI，而不是当前主线 HTTP 接口。

## 3. 当前不作为主线 HTTP 能力承诺的历史模块

以下能力仍可能在仓库的历史测试、服务目录或旧文档中出现，但**不应视为当前主线 Dashboard 的现行产品承诺**：

- Skills 写接口（`/api/skills` 系列）
- Settings Files / Dictionary 写接口
- Outlines split / resplit 写接口
- Edit Assist 写接口
- Codex Bridge 前端拉起写作对话流程

这些能力如需恢复，应以新的产品说明、测试和路由挂载状态为准，而不是以旧文档为准。

## 5. 常见错误码分层

| 状态码 | 常见场景 |
|---|---|
| `400` | 参数非法、请求格式错误 |
| `401` | Basic Auth 缺失或凭据错误 |
| `403` | 路径越界、访问被拒绝 |
| `404` | 项目根不存在、目标资源不存在 |
| `409` | 运行时冲突 |
| `500` | 数据库/文件/服务内部失败 |
| `501` | 运行时能力不可用 |

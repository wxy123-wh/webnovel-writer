# 接口说明（Phase 1 收敛版）

> 本文描述当前主线支持的接口面，并明确这些接口在收敛后分别属于主产品入口还是 companion / ops surface。

## 全局约定

- 主产品形态：**Chat Agent / Chat UI**
- CLI 统一入口：`python -X utf8 webnovel-writer/scripts/webnovel.py <tool> ...`
- Dashboard 默认本地地址：`http://127.0.0.1:8765`
- Dashboard 当前仅保留只读接口
- 统一错误结构：`{ error_code, message, details, request_id }`

## 1. 主入口与伴随入口

### 1.1 主入口

- **Chat Agent / Chat UI**：唯一主产品入口

### 1.2 Companion / Ops Surfaces

- **CLI**：自动化、运维、脚本调用
- **Dashboard HTTP API**：只读观察与诊断

## 2. CLI 接口

CLI 在当前阶段继续存在，但只承担 companion / ops surface 角色。

### 2.1 现有命令面

| 命令面 | 角色 |
|---|---|
| `agent` | 当前 Agent 运行时接口 |
| `codex` | 兼容会话、索引、RAG 命令面 |
| 其他历史命令 | 运维与兼容用途 |

### 2.2 常见命令

| 命令 | 用途 |
|---|---|
| `agent run` | 执行当前 Agent 运行能力 |
| `agent session start/stop` | 管理当前 Agent 会话 |
| `codex session start/stop` | 兼容会话管理 |
| `codex index status` | 查询索引状态 |
| `codex rag verify` | 查询 RAG 验证状态 |
| `dashboard` | 启动只读 Dashboard |
| `init/use/preflight/where` | 项目初始化与运行环境辅助 |

## 3. HTTP API 接口

### 3.1 Dashboard 只读聚合接口

- `GET /health`
- `GET /api/project/root`
- `GET /api/project/info`
- `GET /api/entities`
- `GET /api/relationships`
- `GET /api/chapters`
- `GET /api/scenes`
- `GET /api/reading-power`
- `GET /api/review-metrics`
- `GET /api/dashboard/overview`
- `GET /api/graph`
- `GET /api/files/tree`
- `GET /api/files/read`
- `GET /api/events`
- `GET /api/runtime/profile`

这些接口的职责固定为：**读取、观察、诊断、确认**。

## 4. 不再作为主产品承诺的历史接口面

以下能力即便仍可能在仓库的历史实现、旧测试或旧文档中出现，也不再作为主产品主线承诺：

- Skills 写接口
- Settings 写接口
- Outlines split / resplit 写接口
- Edit Assist 写接口
- Codex Bridge 前端写作对话流程

这些内容如需恢复，必须以新的主产品定义、测试与路由挂载为准。

## 5. 运行时产物

| 模块 | 当前产物 |
|---|---|
| 会话 | `.webnovel/codex/sessions/*` |
| 索引 | `.webnovel/codex/fast-index.json`、`.webnovel/codex/index-state.json` |
| 验证 | `.webnovel/codex/rag-benchmark.json` |
| 迁移 | `.webnovel/migrations/codex-migrate-*.json` |

这些产物在当前阶段继续保留，但它们属于底层运行时，不定义主产品入口。

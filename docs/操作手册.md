# 操作手册

## 1. 环境准备

- Python `>= 3.10`
- Node.js `>= 18`（仅前端构建需要）
- 建议在受信任内网运行 Dashboard

安装依赖：

```bash
python -m pip install -r requirements.txt
```

## 2. 快速启动（首次）

### 2.1 初始化小说项目

```bash
python -X utf8 webnovel-writer/scripts/webnovel.py init ./webnovel-project "你的书名" "题材"
```

### 2.2 绑定工作区

```bash
python -X utf8 webnovel-writer/scripts/webnovel.py use "<PROJECT_ROOT>" --workspace-root "<WORKSPACE_ROOT>"
```

### 2.3 预检

```bash
python -X utf8 webnovel-writer/scripts/webnovel.py --project-root "<PROJECT_ROOT_OR_WORKSPACE_ROOT>" preflight --format json
```

## 3. Chat Agent 写作（主入口）

Chat Agent 是当前唯一主产品入口。用户通过浏览器对话驱动创作。

### 3.1 启动 Chat Agent

```bash
# 先初始化项目（如尚未初始化）
python -X utf8 webnovel-writer/scripts/webnovel.py init ./my-novel "书名" "xuanhuan"

# 启动 Dashboard（内含 Chat Agent UI）
python -X utf8 webnovel-writer/scripts/webnovel.py --project-root "./my-novel" dashboard --host 127.0.0.1 --port 8765
```

浏览器打开 `http://localhost:8765`，默认进入 Chat 页面。

### 3.2 对话写作流程

1. 点击左侧 **"+ 新对话"** 创建新对话
2. 可选：输入对话标题，如"第1章写作"
3. 在输入框输入写作需求，例如：
   - "帮我写第1章的开头，主角出场"
   - "生成第3章的剧情大纲"
   - "检查第2章和第1章的设定是否一致"
4. Agent 会流式返回回复
5. 所有消息自动持久化到 `.webnovel/chat.db`，刷新不丢失

### 3.3 Skill 库管理

1. 在对话界面点击右上角 **"⚡ Skills"** 按钮
2. 侧边抽屉展示所有可用 Skills（来源标记：system / profile / workspace）
3. 点击 **"启用"/"已启用"** 按钮切换 Skill 挂载状态
4. 启用的 Skill 会在当前对话中生效

可用 Skill 来源：
- **system**：内置写作能力（webnovel-write、webnovel-plan、webnovel-review 等）
- **profile**：题材 profile（xianxia、urban、romance、battle 等）
- **workspace**：项目级自定义 skill（`.webnovel/skills/`）

### 3.4 Chat API 直接调用

如需脚本化调用，可直接请求 Chat API：

```bash
# 创建对话
curl -X POST http://localhost:8765/api/chat/chats \
  -H "Content-Type: application/json" \
  -d '{"title": "第1章", "profile": "xianxia"}'

# 发送消息（非流式）
curl -X POST http://localhost:8765/api/chat/chats/{chat_id}/messages \
  -H "Content-Type: application/json" \
  -d '{"content": "帮我写第1章开头"}'

# 发送消息（SSE 流式）
curl -X POST http://localhost:8765/api/chat/chats/{chat_id}/stream \
  -H "Content-Type: application/json" \
  -d '{"content": "帮我写第1章开头"}'

# 查看消息历史
curl http://localhost:8765/api/chat/chats/{chat_id}/messages

# 列出可用 Skills
curl http://localhost:8765/api/chat/skills
```

## 4. CLI 运维命令（companion）

CLI 不再是主写作入口，仅用于自动化、运维和脚本化调用。

```bash
python -X utf8 webnovel-writer/scripts/webnovel.py where
python -X utf8 webnovel-writer/scripts/webnovel.py --project-root "<PROJECT_ROOT>" extract-context --chapter 1 --format text
python -X utf8 webnovel-writer/scripts/webnovel.py --project-root "<PROJECT_ROOT>" status -- --focus all
python -X utf8 webnovel-writer/scripts/webnovel.py --project-root "<PROJECT_ROOT>" codex index status
python -X utf8 webnovel-writer/scripts/webnovel.py --project-root "<PROJECT_ROOT>" codex rag verify --report json
```

## 5. Dashboard 启停

启动：

```bash
python -X utf8 webnovel-writer/scripts/webnovel.py --project-root "<PROJECT_ROOT>" dashboard --host 127.0.0.1 --port 8765
```

常用参数：

- `--no-browser`：不自动打开浏览器
- `--no-bootstrap-index`：不自动初始化缺失索引

停止：前台进程按 `Ctrl+C`。

## 6. 功能操作 SOP

### 6.1 Dashboard 只读观察

1. 打开 Dashboard 查看项目根目录、实体、章节、场景、文件树等只读信息。
2. 若页面提示只读模式，以 CLI / Codex 直接编辑文件为准，不要再按旧 HTTP 写接口操作。
3. 如需确认服务是否正常，优先检查 `/health` 与 `/api/project/root` 返回。

### 6.2 Chat Agent 对话管理

1. 对话列表在 Chat 页面左侧，点击切换。
2. 点击 **"+ 新对话"** 创建新对话。
3. 对话数据存储在 `<PROJECT_ROOT>/.webnovel/chat.db`。
4. 删除对话在对话标题右键菜单或通过 API `DELETE /api/chat/chats/{chat_id}`。

### 6.3 启动写作会话（CLI companion）

```bash
python -X utf8 webnovel-writer/scripts/webnovel.py codex session start \
  --profile battle \
  --project-root "<PROJECT_ROOT>"
```

1. 启动后记录返回的 `session_id`。
2. 在该会话中由 Codex 直接编辑小说文件。
3. 会话结束后显式停止。

### 5.3 查询索引 / RAG 状态

```bash
python -X utf8 webnovel-writer/scripts/webnovel.py codex index status --project-root "<PROJECT_ROOT>"
python -X utf8 webnovel-writer/scripts/webnovel.py codex rag verify --project-root "<PROJECT_ROOT>" --report json
```

说明：`rag verify` 会直接检查连通性，并读取项目自己的 benchmark 产物来判断正确性 / 性能；仓库内 CI 只能证明命令链路和 benchmark 契约可工作，实际商业交付仍应以目标项目自己的报告作为验收依据。

## 6. 迁移、备份与恢复

### 6.1 历史 `.claude` 指针迁移

```bash

python -X utf8 webnovel-writer/scripts/webnovel.py --project-root "<PROJECT_ROOT_OR_WORKSPACE_ROOT>" migrate codex
```

### 6.2 备份与归档

```bash
python -X utf8 webnovel-writer/scripts/webnovel.py --project-root "<PROJECT_ROOT>" backup -- --list
python -X utf8 webnovel-writer/scripts/webnovel.py --project-root "<PROJECT_ROOT>" backup -- --rollback 12
python -X utf8 webnovel-writer/scripts/webnovel.py --project-root "<PROJECT_ROOT>" archive -- --stats
python -X utf8 webnovel-writer/scripts/webnovel.py --project-root "<PROJECT_ROOT>" archive -- --restore-character "<角色名>"
```

## 7. 常见问题排查

| 现象 | 排查建议 |
|---|---|
| `403 workspace_forbidden / workspace_mismatch` | 检查 `workspace_id` 与 `project_root` 是否属于同一工作区 |
| `index.db` 缺失 | 先执行索引相关命令或按启动日志检查自动初始化是否成功 |
| `rag verify` 失败 | 先检查 `.webnovel/vectors.db`、`.webnovel/index.db` 与 `rag_schema_meta` |
| Dashboard 无法访问 | 先检查 `/health` 是否返回 200，以及 `--project-root` 是否指向有效项目 |

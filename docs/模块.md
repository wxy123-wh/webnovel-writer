# 模块说明

> 本文把历史架构文档、SRS 与实施计划收敛为“当前实现优先”的模块视图。

## 1. 总体架构

系统按四层组织：

1. 交互层：CLI（`webnovel.py`）+ Dashboard（FastAPI + React）
2. 服务层：`runtime / skills / dictionary / split / edit_assist / codex_bridge`
3. 数据层：`state.json / index.db / vectors.db` + `.webnovel/*` 专用文件
4. 协作层：Context/Data Agent 与六类 Checker（质量门禁）

## 1.1 仓库目录分层（按模块存放）

为避免根目录混入临时文件，仓库根目录按职责划分为以下模块：

- `webnovel-writer/`：核心源码（CLI、Dashboard、服务与测试）
- `docs/`：对外文档（接口、模块、操作手册）
- `examples/`：示例/沙箱小说工程（本地使用，不参与主仓库提交）
- `__temp__/`：测试与运行临时产物（可周期性清理）
- 其余 `tmp/probe/pytest` 产物：统一视为过期缓存，不应长期停留在根目录

## 2. 核心规则

- 大纲即法律：正文推进要受大纲约束
- 设定即物理：设定冲突必须可检测可处理
- 发明需识别：新增实体要进入结构化状态
- Strand Weave 节奏目标：Quest/Fire/Constellation 约 `60/20/20`

## 4. 核心流程

### 4.1 设定词典抽离流程

1. 读取 `设定集` 文件树与文件内容。
2. 执行 `dictionary/extract`，生成或增量更新条目。
3. 对冲突条目标记并进入人工决策。
4. 决策结果回写词典存储并保留可追溯来源。

### 4.2 双纲拆分与重拆流程

1. `split/preview` 仅生成建议片段，不落盘。
2. `split/apply` 在锁保护下落盘并记录 `idempotency_key`。
3. `resplit/preview` 先计算回退计划再给出新片段。
4. `resplit/apply` 执行回退 + 重拆 + 顺序校验，冲突则阻断写入。

### 4.3 编辑协助流程

1. `preview` 生成改写提案。
2. `apply` 二次校验选区版本（`expected_version/selection_version`）。
3. 成功后落盘并写日志，冲突/失败返回明确错误码。

## 6. 已知限制（来自历史验收与现状）

1. 运行时路径仍存在 `.claude` 兼容读取痕迹，尚未完全收敛为纯 `.codex`。
2. 统一 CLI 下的 `webnovel skill ... / setting ... / outline ...` 子命令尚未完全收口。
3. 全编辑区（设定/大纲/正文）统一 Edit Assist 前端闭环仍在补齐。
4. `split/resplit` 的 p95 性能报告尚未在仓库内沉淀为发布级证据。

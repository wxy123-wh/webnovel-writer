# 模块说明（Phase 1 收敛版）

> 本文描述的是当前仓库在收敛后的**目标模块边界**，以及现有代码与目标模型的映射关系。

## 1. 总体模型

当前项目的唯一主产品形态定义为 **Chat Agent / Chat UI**。因此模块边界不再按“CLI + Dashboard + 插件并列入口”组织，而按“核心能力 + companion surfaces”组织。

目标模型分为两层：

1. `core`：真正定义产品能力的内核
2. `apps`：承载这些能力的界面或运维入口

## 2. 目标模块清单

| 模块 | 目标职责 |
|---|---|
| `core/agent_runtime` | 对话会话、消息编排、上下文装配 |
| `core/skill_system` | skill 安装、卸载、启用、会话挂载 |
| `core/pipeline` | 大纲 → 剧情 → 事件 → 场景 → 整章整合 |
| `core/rag_index` | 索引、检索、验证、一致性检查 |
| `core/project_state` | 项目状态、工作区绑定、运行时元信息 |
| `apps/chat-ui` | 唯一主产品入口 |
| `apps/cli` | 自动化、运维、脚本入口 |
| `apps/dashboard` | 只读观察与诊断面 |
| `apps/vscode-companion` | 编辑器伴随工具 |

## 3. 当前仓库映射

当前仓库尚未物理迁移到上述目录，但现有代码大致映射如下：

| 当前路径 | 当前承载角色 | 目标归属 |
|---|---|---|
| `webnovel-writer/scripts/` | Agent / CLI / 运行时能力 | `core/*` + `apps/cli` |
| `webnovel-writer/dashboard/` | 只读 HTTP API 与前端 | `apps/dashboard` |
| `webnovel-writer/vscode-extension/` | 只读编辑器 companion | `apps/vscode-companion` |
| `webnovel-writer/scripts/codex_skill_profiles/` | 当前 skill/profile 资产 | `core/skill_system` |

## 4. 主线流程

### 4.1 主产品流程

1. 用户进入 Chat Agent / Chat UI
2. 与 Agent 对话
3. Agent Runtime 装配上下文
4. Skill System 选择并挂载会话级 skill
5. Pipeline 推进创作阶段
6. RAG 提供检索与一致性支撑

### 4.2 Companion 流程

1. CLI：执行自动化、运维、状态查询
2. Dashboard：查看只读数据与运行时状态
3. VS Code Companion：浏览项目、辅助跳转

这些流程都不能再反向定义主产品。

## 5. 已知收敛约束

1. CLI 不再是主入口定义。
2. Dashboard 不再是主工作台定义。
3. VS Code 插件不再是主产品形态定义。
4. 历史 Codex / ClaudeCode 心智不再作为产品叙事中心。
5. Pipeline 与 RAG 继续保留，并以内核角色存在。

# Consistency Skill Profile — 一致性检查辅助

> **Profile ID**: `consistency`  
> **适用场景**: 全程生效，贯穿整个连载周期的一致性守卫

## 概述

Consistency Profile 是一个"背景常驻型" profile，不针对特定场景类型，而是为整个连载过程提供一致性保障。长篇网文最致命的问题不是文笔——而是"遗忘"与"矛盾"。

本 profile 通过以下机制保障一致性：

1. **人物性格追踪** — 核心特质不可无故改变
2. **设定体系维护** — 力量/经济/社会规则前后一致
3. **时间线校验** — 事件顺序逻辑自洽
4. **称呼一致性** — 称谓符合人物关系
5. **伏笔追踪** — 伏笔不遗忘、不超时
6. **信息对称管理** — 读者与角色的信息边界清晰

## 文件结构

```
consistency/
├── README.md           # 本文件：Profile 概述与使用说明
├── rules.md            # 一致性检查专项规则（6 条核心约束）
└── check-weights.yaml  # Checker 权重覆盖配置
```

## 加载规则

本 profile 中的 `rules.md` 会在以下时机被加载：

- **会话级加载**: 通过 `codex session start --profile consistency` 启动时加载。建议与题材 profile 叠加使用（如 `--profile consistency,xianxia`）
- **全程生效**: 一致性规则不像 battle/description 那样按场景激活，而是对每一章都生效
- **Checker 权重**: `check-weights.yaml` 将 consistency checker 权重提升至 1.5，强化一致性检测

## Checker 权重说明

| Checker | 默认权重 | 本 Profile 权重 | 说明 |
|---------|---------|----------------|------|
| pacing | 1.0 | **1.0** | 节奏保持标准权重 |
| coolpoint | 1.0 | **0.8** | 一致性优先时爽点适度让步 |
| consistency | 1.0 | **1.5** | 一致性检测大幅强化 |
| hook | 1.0 | **1.0** | 钩子保持标准权重 |

## 适用范围

本 profile 适用于所有题材和场景，建议作为"基础层"始终激活：

- 可与任何 genre profile 叠加
- 可与 battle/description 等 scene profile 叠加
- 独立使用时专注于一致性问题检测

## 常见一致性错误类型

| 错误类型 | 典型表现 | 严重度 |
|---------|---------|--------|
| 人设漂移 | 冷酷角色突然话多且热情 | 🔴 高 |
| 数值矛盾 | 第 5 章说灵石千枚，第 8 章变八百（未解释消耗） | 🔴 高 |
| 时间错乱 | 两天内发生了三天的事 | 🟡 中 |
| 称呼混乱 | 下属对上司从"大人"突然变"老兄" | 🟡 中 |
| 伏笔遗忘 | 第 2 章埋的伏笔到第 50 章仍无呼应 | 🟡 中 |
| 信息泄露 | 角色知道只有读者应该知道的信息 | 🔴 高 |
